## 1 Введение

Проект называется «Доставка стройматериалов».
Данный проект представляет собой мобильное приложение, которое позволяет пользователям приобрести необходимые строматериалы и заказать дотавку по необходимому адресу.

<a name="11"/>

### 1.1 Назначение
В этом документе описаны функциональные и нефункциональные требования к приложению «доставка стройматериалов» для ОС Windows. Этот документ предназначен для команды, которая будет реализовывать и проверять корректность работы приложения.

<a name="12"/>

### 1.2 Бизнес требования

<a name="121"/>

#### 1.2.1 Исходные данные

Процесс доставки строительных материалов часто связан с организационными сложностями, такими как необходимость выбора нужных товаров, оформления заказа, оплаты и отслеживания доставки. У многих компаний возникает потребность в мобильных приложениях для удобного управления этими процессами.

<a name="122"/>

#### 1.2.2 Возможности бизнеса
С развитием цифровых технологий и мобильных устройств, компании всё чаще стремятся использовать приложения для упрощения взаимодействия с клиентами. Приложение для доставки строительных материалов может повысить скорость обработки заказов, уменьшить количество ошибок и улучшить клиентский опыт, предлагая удобный интерфейс для выбора товаров, оформления и оплаты заказа.

<a name="123"/>

#### 1.2.3 Границы проекта
Приложение будет включать каталог строительных материалов, возможность оформления заказа, отслеживание статуса доставки. Оно будет предназначено для конечных пользователей — клиентов, заинтересованных в быстром и удобном процессе покупки строительных материалов.

<a name="13">

### 1.3 Существующие аналоги
На рынке уже существуют мобильные приложения для доставки товаров, такие как "OZON" и "Wildberries", которые предлагают схожие функции, но не ориентированы исключительно на строительные материалы и не поддерживают большие объёмы товара.

<a name="2"/>

## 2 Требования пользователя

<a name="21"/>

### 2.1 Программные интерфейсы
Приложение будет использовать Firebase для различных функций:
Firebase Authentication для аутентификации пользователей.
Firebase Realtime Database для хранения данных о заказах, товарах и пользователях.
Firebase Storage для хранения изображений строительных материалов.

### 2.3 Характеристики пользователей

<a name="231">

#### 2.3.1 Классы пользователей
В описываемом приложении есть два класса пользователя — обычный пользователь и администратор. 

<a name="232">

#### 2.3.2 Аудитория приложения
Приложение рассчитано на широкую аудиторию, в основном пользователей, занимающихся строительством или ремонтом. Уровень образования не имеет значения, а для использования приложения требуется базовый уровень технической грамотности и умение пользоваться мобильными приложениями для поиска, выбора и оформления заказов.

<a name="24"/>

### 2.4 Предположения и зависимости
Приложение предполагает стабильную работу мобильного устройства и доступ к интернету для взаимодействия с базой данных Firebase. Пользователи будут регистрироваться через Firebase Authentication, а данные о товарах, заказах и изображениях будут храниться в Firebase Realtime Database и Firebase Storage.

## 3 Системные требования

<a name="31"/>

### 3.1 Функциональные требования

<a name="311">

#### 3.1.1 Авторизация
При входе в приложение пользователь должен зарегистрироваться или зайти в свой аккаунт

| Функция | Требование |
|:---|:---|
| Регистрация нового пользователя | Приложение запрашивает имя, почту и пароль. Пользователь вводит данные и регистрируется |
| Вход с авторизацией | Приложение запрашивает почту и пароль, после чего пользователь входит в профиль |

<a name="312">

#### 3.1.2 Просмотр доступных строительных материалов и добавление новых
На странице каталога пользователь может перемещаться по необходимым категориям товаров.

| Функция | Требования | 
|:---|:---|
| Выбор каталога от имени администратора | После нажатия на каталог администартор переходит на страницу где заполняет форму товара для размещения его в каталоге |
| Выбор каталога от имени пользователя | После нажатия на каталог пользователь переместиться на страницу со списком товаров этой категории |

<a name="313">

#### 3.1.3 Просмотр карзины товаров 
У пользователя есть возможность оформить доставку.

| Функция | Требования | 
|:---|:---|
| Оформление доставки | Пользователь оформляет доставку товаров корзины указывая адресс доставки |

<a name="314">

#### 3.1.4 Просмотр статуса доставки
У пользователя есть возможность просмотреть статус доставки

<a name="315">

#### 3.1.5 Меню пользователя
Пользователь может редактировать свои данные, выходить с аккаунта или удалить его.

| Функция | Требования | 
|:---|:---|
| Редактирование почты | Пользователь имеет возможность редактировать почту привязанную к аккаунту |
| Редактирование имени | Пользователь имеет возможность редактировать имя привязанное к аккаунту |
| Редактирование номера телефона | Пользователь имеет возможность привязать и редактировать номер телефона привязанный к аккаунту |
| Выход из аккаунта | Пользователь имеет возможность выйти из аккаунта |
| Удалить аккаунт | Пользователь имеет возможность удалить аккаунт |

<a name="32"/>

### 3.2	Нефункциональные требования

1. Надежность: Приложение должно работать корректно при стабильном подключении к Firebase. Должна быть предусмотрена обработка ошибок, связанных с потерей соединения или некорректной работой базы данных.
2. Безопасность: Приложение должно обеспечить защиту данных пользователей через безопасную авторизацию в Firebase. Все данные, передаваемые между клиентом и сервером, должны быть защищены.
3. Производительность: Приложение должно загружать материалы быстро, с минимальными задержками, оптимально используя ресурсы устройства и обеспечивая плавный опыт взаимодействия для пользователей.
